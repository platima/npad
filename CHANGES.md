# Changes

Commit-level history of npad, one entry per commit, grouped by version.
Curated per-release notes live in CHANGELOG.md; this file is the raw record.

Maintenance: every commit appends its entry to the current version's section
(newest first). The entry for the commit being made carries no hash, since
the hash is unknown until the commit exists; it may be added retroactively.
Version numbers follow semver (0.x: minor = feature rounds / breaking
changes, patch = fix-only rounds). Mapping of pre-0.2.0 commits to versions
is a best-effort reconstruction from the changelog and commit subjects.
